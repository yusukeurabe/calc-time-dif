# Time zone timeline — 設計書

作成日: 2026-06-12
ステータス: ユーザー承認済みデザインに基づく設計

## 1. 概要

2都市間の時差を視覚的・直感的に変換できるウェブアプリ。
デフォルトは「日本（東京）↔ カナダ・バンクーバー」。

- 主用途: 「日本の◯時はバンクーバーだと何時？」という**時刻変換**
- 横に並んだ2本のタイムラインバーで、同じ瞬間が縦に揃う
- カーソルのドラッグ／タップ／時刻入力のどれでも変換できる
- スマホ・PC両対応（レスポンシブ）、Vercel で公開

## 2. 画面仕様（承認済みモックアップ v5 準拠）

### 2.1 ヘッダー
- アプリ名: Time zone timeline
- 時差バッジ: 例 "Vancouver is 16 hours behind Japan"
  - 選択中のカーソル時刻時点での時差を表示（夏時間で16時間/17時間が切り替わる）
  - 同時刻なら "Both cities share the same time"（JA: 「2都市は同じ時刻です」）、相手が進んでいる場合は "ahead"
- 言語切替: EN / 日本語 のトグル。**デフォルトは英語**。選択は localStorage に保存

### 2.2 コントロール行
- 都市セレクト × 2（基準都市・相手都市）
- 入れ替えボタン（⇄）
- 日付入力（カレンダー）: 指定した日付へタイムラインを**ジャンプ**するショートカット
  - 値はカーソル位置（基準都市側の日付）に常に追従して表示される
- Now ボタン: 現在時刻の位置までスクロールを戻し、カーソルも現在時刻へ移動

### 2.3 タイムライン（中核UI）
- 上下2本のバー。上=基準都市、下=相手都市。**同じ瞬間が同じX座標**に揃う
- 1時間 = 1マス（固定幅 28px）。マスの中にその都市の**ローカル時刻の数字（0〜23）**を表示
- 昼（6:00〜17:59）= アンバー系、夜（18:00〜5:59）= パープル系の背景色
- 各バーの直下に**日付帯**: その区間の日付＋曜日（例 "Fri, Jun 12"）を表示
  - ローカル0時の位置に**太い縦の区切り線＋"0:00"ラベル**
  - 「今日」（現在日付）の帯セグメントは青系でハイライト
- **横スクロールで実質無限に前後の日へ移動できる**
  - 仮想スクロール方式: 表示範囲＋前後バッファ分のみDOMを生成し、スクロールに応じて追加・破棄
- 黒い選択カーソル（縦線＋上部のつまみ）: タイムラインと一緒にスクロールする
  - ドラッグまたはバー上のタップ/クリックで移動。**15分刻み**にスナップ
- 緑の破線「Now」マーカー: 現在時刻の位置に常時表示（1分ごとに更新）
- 初期表示: カーソル＝現在時刻、スクロール位置はカーソルが画面内に見える位置
- 都市名ラベル（Japan / Vancouver）は position:sticky で左端に固定

### 2.4 結果カード（2枚）
- 各都市の「都市名 · 曜日, 日付」と**24時間制**の時刻を大きく表示
- 昼は太陽アイコン、夜は月アイコン
- 基準都市と日付がずれる場合はバッジ表示: "Previous day" / "Next day"

### 2.5 直接時刻入力
- 基準都市の時刻を `<input type="time">` で直接入力 → カーソルがその時刻へ移動（必要ならスクロールも追従）

### 2.6 レスポンシブ
- マスは固定幅のまま、狭い画面では横スクロールでカバー（モバイルはスワイプ操作が自然に効く）
- コントロール行は折り返し、結果カードは1列に積む

## 3. 都市リスト

内蔵リスト（約20都市、IANAタイムゾーンID使用）:

## 4. 技術構成

| 項目 | 採用 | 理由 |
|---|---|---|
| フレームワーク | なし（HTML + CSS + vanilla JS） | この規模に十分。ビルド不要で壊れにくい |
| 時差計算 | ブラウザ内蔵 Intl API（IANA TZ DB） | 夏時間を含め正確。外部ライブラリ不要 |
| 保存 | localStorage | 都市の組合せ・言語設定を次回復元 |
| 公開 | Vercel（静的サイト） | URL公開、設定ほぼ不要 |

### ファイル構成
```
calc-time-dif/
├── index.html        … ページ骨格
├── style.css         … スタイル（ライト/ダーク対応は不要、ライトのみ）
├── src/
│   ├── app.js        … UI制御・描画・イベント（エントリポイント）
│   ├── timeline.js   … タイムラインの仮想スクロール描画
│   ├── timezone.js   … 時差計算の純ロジック（DOM非依存）
│   ├── cities.js     … 都市データ（名前EN/JA、IANA ID）
│   └── i18n.js       … UI文言（EN/JA）
├── tests/
│   └── timezone.test.js … 変換ロジックのユニットテスト（node:test）
└── docs/superpowers/specs/ … 本設計書
```
※ モジュールは ES Modules（`<script type="module">`）。ビルドステップなし。

## 5. 状態とデータフロー

アプリの状態は1つのオブジェクトに集約:
```
state = {
  cityA,        // 基準都市ID（上のバー） 初期値: tokyo
  cityB,        // 相手都市ID（下のバー） 初期値: vancouver
  cursorUTC,    // 選択中の瞬間（UTCミリ秒） 初期値: 現在時刻
  lang,         // 'en' | 'ja' 初期値: en
}
```
- どの操作（ドラッグ／タップ／時刻入力／日付ジャンプ／Now／都市変更）も `cursorUTC` などの状態を更新 → 1つの `render()` がバッジ・カーソル位置・結果カード・時刻入力欄をすべて再描画
- タイムラインの原点は UTC 瞬間。各都市の表示は Intl でローカル時刻に変換するため、**両バーの整列・日付境界・夏時間が自動的に正しくなる**

### 夏時間（DST）の扱い
- バンクーバーの夏時間切替日は、ローカルの1日が23時間/25時間になる。バーのマスは「実際に存在するローカル時刻」を Intl から求めて生成するため、切替日はマス数が23/25個になり、ズレは生じない
- 時差バッジはカーソル時点の両都市オフセット差から算出

## 6. エラー処理

- localStorage の値が不正・古い → 黙ってデフォルト（日本↔バンクーバー、EN）に戻す
- `Intl.DateTimeFormat` 非対応の極端に古いブラウザ → 「お使いのブラウザは対応していません」メッセージのみ表示
- 入力系は選択式・time/date input のみで、自由入力がないため不正入力の余地はほぼない

## 7. テスト

`node --test` によるユニットテスト（DOM 非依存の timezone.js が対象）:
- JST↔PDT（夏: −16時間）、JST↔PST（冬: −17時間）の変換
- 日付境界をまたぐ変換（Previous day / Next day 判定）
- DST切替日（2026-03-08, 2026-11-01）前後の変換と「1日のマス数」
- 日付帯ラベル・曜日の生成（EN/JA）
- 時差バッジの文言生成（behind / ahead / same time）

UI はブラウザでの手動確認（スマホ幅・PC幅、スクロール、ドラッグ、各ボタン）。

## 8. スコープ外（今回はやらない）
